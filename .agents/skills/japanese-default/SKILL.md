# 日本語ベース言語

このスキルは、ユーザ向けの出力全体を日本語で統一したいときに使います。

## 使うタイミング

次のような依頼があれば、このスキルを使います。

- 「日本語でお願いします」「以後は日本語で返答して」など、会話の基準言語を日本語にしたい
- 生成する文書、テンプレート、説明文、レポート、議事録なども日本語にしたい
- ファイルに書き出す内容だけでなく、途中の確認や質問、進捗共有も日本語にしたい
- 英語の情報を扱いつつ、ユーザに見せる最終成果物は日本語にしたい

## 基本方針

1. ユーザへの返答、確認、要約、進捗報告、提案は日本語で行います。
2. 新規に作るファイルのうち、ユーザ向けの文章、見出し、ラベル、説明文は日本語で書きます。
3. 既存ファイルを編集する場合も、ユーザが日本語化を求めている範囲では、追加・変更するユーザ向け文面を日本語にします。
4. ユーザがあとから別の言語を明示したら、その最新の指示を優先します。

## 日本語にする対象

- 会話中の説明、質問、確認
- 計画、進捗報告、レビューコメントの本文
- ドキュメント、手順書、テンプレート、レポート
- 画面文言、ラベル、ヘルプ文、プレースホルダーなどのユーザ向けテキスト
- ファイルに出力する要約、説明、注記、見出し

## 原文を保つ対象

次の要素は、正確さや互換性を優先して原文を保ちます。必要なら直後に日本語で短く補足します。

- ソースコード、識別子、関数名、クラス名、変数名
- コマンド、CLI 出力、ログ、エラーメッセージ
- API 名、HTTP ヘッダー名、設定キー、環境変数名
- URL、パス、スキーマ、JSON キーなど固定の構造
- 引用文、既存の固有名詞、外部仕様上そのまま使う必要がある文言

## 迷ったときの進め方

1. まずユーザ向けの文章を日本語にそろえます。
2. そのままでは意味や互換性が崩れる要素だけ原文を残します。
3. 日本語化の範囲が曖昧なら、短い確認を日本語で行います。
4. 英語などの原文を残した場合は、必要に応じて日本語で要点を添えます。

## 文体

- 自然で読みやすい日本語を使います。
- 不必要な直訳を避け、意味が伝わる表現を選びます。
- 専門用語は無理に訳し切らず、必要なら原語を括弧書きで併記します。
