# 作成手順

## 手順
1. 変更内容から主目的を 1 つに要約する
1. 8 種類の prefix から最も適切なものを選ぶ
1. 主目的が新機能か、不具合修正かを切り分ける
1. 変更内容を日本語で簡潔な subject にする
1. 本文に `概要` と `Why` を書く
1. 長くなる箇所は次の箇条書きへ分ける
1. 体言止め中心で整える
1. 1行目を `<type_>: <subject>` の形にする
1. 2行目を空行にする
1. prefix が 5 文字固定か確認する

## 本文テンプレート
```text
<type_>: <subject (日本語)>

- 概要:
  - 何を変えたか
  - 必要なら次の要点
- Why:
  - なぜこの対応を行なったか
  - 必要なら背景や制約
```

## 書き方の要点
- 一文が長くなりそうなら、次の箇条書きへ分ける
- 1 つの箇条書きには 1 論点だけを書く
- 本文も日本語で簡潔に書く
- 長文より短い箇条書きを優先する
